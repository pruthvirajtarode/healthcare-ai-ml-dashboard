## File Descriptions

### Core Files

#### 1. **run_pipeline.py** âœ“
- **Purpose:** Master execution script
- **Function:** Runs all steps sequentially (cleaning â†’ EDA â†’ features â†’ ML)
- **Usage:** `python run_pipeline.py`
- **Output:** Generates all processed data files and reports

#### 2. **config.py** âœ“
- **Purpose:** Centralized configuration
- **Contains:** 
  - Data quality rules
  - Model parameters
  - Risk classification rules
  - API settings
- **Usage:** Imported by other modules

#### 3. **requirements.txt** âœ“
- **Purpose:** Python package dependencies
- **Packages:**
  - pandas, numpy, scikit-learn
  - matplotlib, seaborn (visualization)
  - flask, flask-cors (API)
- **Usage:** `pip install -r requirements.txt`

### Backend Scripts

#### 4. **scripts/data_cleaning.py** âœ“
- **Purpose:** Data quality and cleaning
- **Key Functions:**
  - `load_data()` - Load CSV file
  - `handle_missing_values()` - Fill NaN values
  - `handle_invalid_ages()` - Remove age > 100
  - `handle_invalid_bmi()` - Fix negative BMI
  - `handle_invalid_glucose()` - Fix negative glucose
  - `handle_invalid_blood_pressure()` - Fix extreme BP values
  - `clean_data()` - Execute full pipeline
- **Output:** `healthcare_data_cleaned.csv`

#### 5. **scripts/eda_analysis.py** âœ“
- **Purpose:** Exploratory Data Analysis
- **Key Functions:**
  - `get_basic_statistics()` - Dataset overview
  - `analyze_age_distribution()` - Age analysis
  - `analyze_bmi_distribution()` - BMI analysis with categories
  - `analyze_blood_pressure()` - BP analysis and categorization
  - `analyze_glucose()` - Glucose analysis
  - `analyze_disease_risk()` - Disease distribution
  - `analyze_geographical_distribution()` - City analysis
  - `analyze_correlations()` - Feature correlations
  - `identify_high_risk_patients()` - Risk identification
  - `perform_eda()` - Execute full EDA
- **Output:** `eda_report.json`, `high_risk_patients.csv`

#### 6. **scripts/feature_engineering.py** âœ“
- **Purpose:** Feature creation and KPIs
- **Key Features Created:**
  - Health Score (0-100)
  - Risk Score (0-100)
  - Age Groups (5 categories)
  - BMI Categories (4 categories)
  - Metabolic Health Score
  - Cardiovascular Risk Score
  - Glucose-BMI Ratio
  - Disease Risk Priority
  - City Encoding
  - Interaction Features (Age-BMI, BP-Glucose)
- **Key Functions:**
  - `create_health_score()` - Composite health metric
  - `create_risk_score()` - Comprehensive risk assessment
  - `create_*_features()` - Various feature creation methods
  - `calculate_kpis()` - Calculate all KPIs
  - `engineer_features()` - Execute full pipeline
- **Output:** `healthcare_data_engineered.csv`, `kpi_report.json`

#### 7. **scripts/ml_preparation.py** âœ“
- **Purpose:** ML model training and evaluation
- **Key Functions:**
  - `prepare_features_and_target()` - Feature selection
  - `handle_class_imbalance()` - Balance classes
  - `split_data()` - Train/test split
  - `scale_features()` - Normalize features
  - `train_baseline_models()` - Train LR and RF models
  - `get_model_prediction_example()` - Example predictions
  - `prepare_ml_dataset()` - Execute full pipeline
- **Models:**
  - Logistic Regression
  - Random Forest (100 estimators)
- **Output:** `ml_preparation_report.json`, `ml_training_data/`

#### 8. **app.py** âœ“
- **Purpose:** Flask API backend
- **Endpoints:**
  - `/` - Serve dashboard
  - `/api/health` - Health check
  - `/api/eda-report` - EDA results
  - `/api/kpi-report` - KPI metrics
  - `/api/ml-report` - ML model info
  - `/api/dataset-stats` - Dataset statistics
  - `/api/high-risk-patients` - High-risk patients
  - `/api/distribution/<metric>` - Distribution data
  - `/api/predict-risk` - Risk prediction endpoint
  - `/api/patient-profile/<id>` - Patient details
  - `/api/summary` - Overall summary
- **Usage:** `python app.py` (runs on port 5000)

## How to Use This Project

### For Data Analysis
1. See `README.md` for complete documentation
2. Run `python run_pipeline.py` to execute all steps
3. Review generated reports in `data/` folder

### For Dashboard
1. Open `frontend/index.html` in browser
2. View different tabs and charts
3. Use prediction tool to assess risk

### For ML Models
1. Check `ml_preparation_report.json` for model details
2. Use `app.py` to deploy as API
3. Models use Random Forest (best accuracy)

### For Integration
1. Use Flask API (`app.py`) for backend integration
2. Modify `frontend/` files for custom styling
3. Adjust `config.py` for different parameters
